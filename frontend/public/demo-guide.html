Firewall prerequisite for live Kaltura avatar WebRTC:

Whitelist TURN server IP range 3.41.177.176/29 (IPs 3.41.177.176 – 3.41.177.183) on UDP/TCP 443 and UDP/TCP 80 outbound. Without this rule, the WebRTC connection will fail behind corporate NAT/firewalls. For local testing on a home network this is usually not needed.

1 Start the Avatar Session

  1. a
    Open https://localhost:5173 You'll see a two-panel layout — avatar on the left, form area on the right.
  2. b
    Click "Start Session" in the avatar panel The backend calls Kaltura with your KS, gets a sessionId + JWT token, and the Kaltura SDK establishes a WebRTC connection. The avatar video appears.
  3. c
    Wait for the status dot to turn green ("Live") This means WebRTC is established and the avatar is ready to speak.

2 Load the Discharge Form

  1. a
    Click "Load Discharge Form" The React frontend opens a POST SSE connection to /api/agent/discharge-form/stream, which proxies to the Gemini-powered ADK agent on :8000.
  2. b
    Watch the form build progressively (4 waves) Form title and section headers appear instantly after Wave 1. Patient fields, discharge fields, medications, and follow-up load in sequence — each wave is a separate A2UI surfaceUpdate message.
  3. c
    Listen — Alex narrates each section as it appears Each wave embeds a KalturaAvatarView component with a spokenText literalString. As the component arrives, the frontend calls session.sayText() and the avatar speaks the guidance aloud.

🔊 Avatar speech timeline

WaveWhat appears on screenWhat Alex says
Wave 1 Form title + 4 section headers "Welcome to your digital discharge process. I'm Alex, your discharge guide…"
Wave 2 Patient name, DOB, MRN, admission date, physician, diagnosis fields "Please fill in your patient details. Make sure your name, date of birth…"
Wave 3 Discharge date, condition, destination, notes + medication list "Now let's review your discharge details and medication schedule…"
Wave 4 Follow-up date, physician, instructions, acknowledgment checkbox, submit button "Almost done. Please confirm your follow-up appointment…"
Submit Form resets "Thank you. Your discharge form has been submitted. A member of the care team will be with you shortly."

3 Fill in the form & submit

  1. a
    Enter patient data in any field All fields write to a local data model in real-time (A2UI's "Write contract"). No network call until the form is submitted.
  2. b
    Select discharge condition and destination MultipleChoice pill buttons — click to select one option.
  3. c
    Review the pre-populated medication (Lisinopril) Rendered via a template-bound List — one Card per item in /medications array.
  4. d
    Check the acknowledgment box Boolean binding to /acknowledgment/confirmed.
  5. e
    Click "Confirm Discharge" A2UI fires the confirm_discharge action with the full data model as context. Alex speaks a confirmation. The form resets and is ready for the next patient.

🛠 Troubleshooting

SymptomFix
Avatar video never loads / "Connection failed" Check TURN IP range 3.41.177.176/29 is whitelisted on UDP/TCP 443+80. Ensure you're on HTTPS (not HTTP).
Certificate warning on first open Expected for self-signed dev cert. Click Advanced → Proceed to localhost.
Form never loads / "event: error" in console ADK agent may be down. Check http://localhost:8000/list-apps — should return ["agent","routes"].
Gemini 429 quota errors Free-tier daily limit hit. Switch model in backend/agent/agent.py or wait ~1 hour.
CORS error in browser console Ensure backend/.env ALLOWED_ORIGINS includes https://localhost:5173.
"KALTURA_KS is not set" on backend start Add your real Kaltura Session key to backend/.env. Get it from your Kaltura representative.
Alex doesn't speak Avatar session must be in READY state before form loads. Start Session first, then Load Discharge Form.
